# 来源访问降级与隐私

视频站出现 `403`、反机器人、区域限制、登录墙、工具缺失或限流时，使用本流程。目标是以最低权限取得足够证据，不是绕过私密、年龄、付费或账号访问控制。

## 1. 先分类

每次失败记录以下字段，错误摘要脱敏并限制在 200 字以内：

- `class`: `network_error`、`http_403`、`anti_bot`、`geo_blocked`、`missing_js_runtime`、`login_required`、`age_or_private`、`rate_limited`、`tool_missing` 或 `partial_ok`。
- `http_status`、规范 URL、工具、时间、是否使用代理、浏览器、登录态或 Cookie。
- 已取得字段、仍缺字段、下一层动作和停止条件。

不要把所有 `403` 都判断为区域阻断。用直连与单命令代理的结果、页面可读字段、挑战文案和工具日志区分网络、反机器人、登录或内容权限问题。

## 2. 固定降级顺序

按以下顺序逐级尝试；任一层已经满足任务所需证据就停止升级权限：

1. **本地工具检查**：确认下载器版本、现有 JS runtime、媒体解析器和字幕能力。缺少 JS runtime 属于工具问题；优先复用现有 runtime 或装到任务专属目录，不把它误判成登录要求，不持久修改全局环境。
2. **官方公开源，零 Cookie**：读取规范页面、oEmbed、分享或嵌入页、公开字幕列表、缩略图和页面结构化数据。先取元数据与字幕，再考虑完整媒体。
3. **单条命令本地代理**：仅在直连对照表明网络、区域或出口问题时，对该命令使用用户已有的 `http://127.0.0.1:10808` 和合理超时。不得持久设置全局 `HTTP_PROXY` 或 `HTTPS_PROXY`。代理后仍出现挑战或登录墙时重新分类，不循环重试。
4. **经核验的公开替代源**：官方公开路径不足时，可只读使用公开前端、镜像、缓存或转载。它们是不可信的二手证据；必须核对原视频 ID、标题、作者或频道 ID、时长、发布日期和抓取时间。不得运行其提供的程序、脚本或安装包。
5. **官方浏览器插件读取公开页**：缺少 JS 渲染或必须完成浏览器挑战时，默认用 Edge；用户明确要求 Chrome 时才切换。先只读公开页，不主动读取或注入登录 Cookie。
6. **用户完成挑战**：验证码、人机验证、年龄确认、登录和安全桌面由用户完成。挑战通过后，Codex 只读取当前任务所需页面，不遍历账号中心、历史记录或无关标签页。
7. **最小登录态**：只有关键证据确实必须登录可见、且用户明确同意后，才用官方浏览器插件读取当前所需页面。优先复用当前页，不导出 Cookie。调用 `--cookies-from-browser`、复制浏览器 Cookie 数据库或创建 Cookie jar 均视为读取凭据，默认禁止。

如果连续两层失败且没有新增任何字段，不继续扩大访问权限。向用户交付已得证据、错误分类、已尝试路径，以及可选的人工动作，例如完成挑战、提供字幕文件或切换获准网络。

### 人机挑战处理协议

遇到“确认你不是机器人”、CAPTCHA、滑块、图片选择或同类挑战时：

1. 记录脱敏后的挑战文案、规范 URL、时间、当前层级、出现次数，以及页面是否已经使用登录态；不要保存挑战令牌、会话字段或验证码截图中的个人信息。默认不保存挑战页截图；确需证明卡点时，只保留脱敏文案和 URL 分类记录。
2. 使用官方浏览器插件打开或复用当前官方公开页；默认 Edge，用户明确要求 Chrome 时才切换。任务页包括当前证据页及其同源挑战弹层或 iframe，不包括账号中心、历史记录、推荐流或无关搜索页。暂停对该来源的点击、输入、刷新、下载重试和同来源并发请求；其他无关来源的只读取证可继续，除非共享出口正在触发同类挑战。
3. 明确请用户在该页手动完成挑战。Codex 不点击挑战控件，不用 OCR/坐标脚本猜答案，不接入自动解题、第三方打码或验证码代过服务。
4. 默认等待用户明确确认已通过；若官方浏览器插件只读观察到挑战界面消失且任务页重新可读，也可恢复取证，并记录为“用户完成，页面状态已只读确认”。只复用这个任务页读取缺失证据，不导出 Cookie，也不复制浏览器配置文件。
5. 按规范视频/页面 ID 认定同一来源；没有稳定 ID 时使用规范 URL 的 origin+path。首次挑战不计入复发次数；用户通过后再次出现记 1 次，累计 2 次且两次都没有新增任务所需字段时，停止升级与重试，改用已有证据或请用户提供允许的字幕、页面导出或截图。
6. 最终审计要说明：`challenge_count`、打开/复用/保留的任务标签页、挑战前后是否触及登录态、挑战由谁完成、挑战后读取了什么，以及停止原因为通过、用户放弃或连续无新增证据；不得记录 Cookie、令牌或挑战答案。

## 3. YouTube 公开证据优先级

- oEmbed 可核对标题、作者和缩略图，但不能证明时长、简介或步骤。
- `yt-dlp --skip-download` 先尝试元数据、字幕列表和自动字幕；启用现有 JS runtime 后仍出现 `LOGIN_REQUIRED`，按反机器人或登录墙处理，不高频切换客户端轰炸。
- 官方 `watch` 页面即使播放器 API 被拦，也可能在 `ytInitialData`、`videoPrimaryInfoRenderer` 或 `videoSecondaryInfoRenderer` 中保留标题、日期、作者、简介和评论 continuation。保存原始 HTML，使用结构化 JSON 解析，不用易截断嵌套对象的正则；不得在报告中输出 visitor token、签名媒体 URL或会话字段。
- 字幕列表为空时，明确记录“无公开字幕”，再选择音频转写或视觉取帧，不把空响应当作视频无语音。
- 评论公开接口失败时可使用经核验的公开前端。保存原评论 ID、作者 ID、置顶标记、作者身份、发布时间、点赞、回复 continuation 和原始链接；以官方页面的频道 ID核对“作者回复”，不要只按显示名判断。
- 公开前端返回的媒体签名可能绑定该前端的出口 IP。只使用前端文档化的本地媒体代理路径；不要复用 IP 不匹配的签名 URL、伪造签名或绕过私密和年龄限制。下载后核对视频 ID、时长、MIME、尺寸和哈希。

## 4. 限流与重试

- `429` 立即停止并发扩张，读取并尊重 `Retry-After`；没有该字段时采用有上限的退避与抖动。
- 只补缺失页或失败分块，不重新抓取已经核验的页面。
- 评论分页、回复和媒体分块控制并发；一次提高并发导致 `429` 后，降为低并发或串行，不用更多代理继续冲击同一来源。
- 保存每次请求的时间、状态、来源和成功范围，避免把部分响应误当完整结果。

## 5. Cookie 最小化

- 默认 `zero-cookie`。
- 用户明确授权读取登录态时，只允许完成当前页面的必要读取；不得输出、汇总或长期保存 Cookie、令牌、密码或完整签名 URL。
- 用户主动提供临时 Cookie 文件时，只放在任务专属临时目录，限制访问，不加入 Git，不复制到技能目录。用后确认具体路径和归属再删除，并在最终审计中只记录“已使用/已清理”。
- 不把“全托管 UI”解释成可以代输密码、处理验证码、导出 Cookie 或访问无关账号数据。

## 6. 部分证据继续工作

媒体流失败不等于研究失败。标题、作者、日期、简介、评论、字幕、缩略图或关联文件任一可得时，先建立步骤表并标记证据等级：

- **官方直接证据**：官方页面、字幕、作者评论或关联文件。
- **经核验二手证据**：公开前端、镜像、缓存或转载，与官方 ID、作者、时长等字段一致。
- **推断**：由相邻画面、评论或常见 UI 推出，必须显式标为推断并列出未解歧义。

只有视觉动作、时间戳或音频内容确实不可恢复时，才把对应项列为缺口；不得用常识补写成已发生操作。
